Line break and whitespace

Posted on 2019-11-05 Edited on 2019-11-23 In Miscellenous

Line break

A.k.a newline is a character/characters denoting the end of a line. Basically, there’re two types of line break:

  • Unix-like line break(LF, line feed), in ASCII encode, \n.
  • Windows-like line break(CRLF, carriage return line feed), in ASCII encode, \r\n.

In practice, most modern text editors allow both types of line breaks, usually by pressing the enter/return key on the keyboard.

There is another thing worth illustrating here, when you start a new line, what you actually do is starting a new paragraph because most text editor support word worp, which means the text you see on the screen will automatically change accoring to the size of the display window. Therefore, there is no need to manually start a new line if you just what to keep on typing. The result is that every time you press enter, the computer will interpretes it as a hard enter, in other words, you press enter to start a new paragraph instead of a new line. In occasions where you do need to start a new line (e.g. in poetry, list, address), a soft enter is needed, which is usually done either by explicitly inserting a “\
“ sign or by press shift + enter.

Whitespace

In markdown/HTML, sequential whitespace will be rendered as a single whitespace, if you want to insert consecutive whitespaces, you can use html &nbsp sign. (nbps means non-breaking space)
